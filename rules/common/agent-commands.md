# 에이전트 자동 전환

사용자의 요청 의도에 따라 자동으로 적절한 작업 모드를 선택합니다.

## 계획 수립 (Planning)

다음과 같은 요청이 감지되면 계획 수립 모드로 동작합니다:
- "계획 세워줘", "플래닝 해줘", "설계해줘", "분석해줘", "어떻게 구현할지", "방향 잡아줘"
- "plan", "blueprint", "design", "how to implement"

### 계획 수립 모드 동작

1. 코드를 바로 작성하지 않는다
2. 단계별로 사고를 정리하며 진행한다 (sequential thinking)
3. rules에 정의된 프로젝트 구조와 패턴을 기반으로 계획한다 (불필요한 코드베이스 탐색 금지)
4. 구현에 직접 관련된 파일만 필요 시 확인한다
5. 영향 범위와 리스크를 분석한다
6. 결과를 `plans/{name}.md`에 저장한다

### 대규모 계획 (여러 PR 규모)

- 3~12개 PR 단위 스텝으로 분해
- 각 스텝에 브랜치명, 태스크, 검증 커맨드, 완료 조건, 의존 관계 포함
- 병렬 실행 가능 여부 표시
- 참조: blueprint, planning-template, deployment-patterns

### 단일 기능 설계 (1 PR 규모)

- 변경 필요 파일/모듈 식별
- 구현 접근법과 사이드이펙트 예측
- 참조: planning-template, api-design, codebase-onboarding

## 코드 생성 (Generation)

다음과 같은 요청이 감지되면 코드 생성 모드로 동작합니다:
- "만들어줘", "구현해줘", "코드 짜줘", "추가해줘", "수정해줘"
- "create", "implement", "generate", "add", "fix"

### 코드 생성 모드 동작

- 기존 코드 패턴과 네이밍 컨벤션을 따른다
- 최소한의 변경으로 요구사항을 충족한다
- 프로젝트 디렉토리 구조를 준수한다

## 코드 리뷰 (Review)

다음과 같은 요청이 감지되면 리뷰 모드로 동작합니다:
- "리뷰해줘", "검토해줘", "코드 봐줘"
- "review", "check"

### 리뷰 모드 동작

- `git diff`로 변경 사항 확인
- CRITICAL/HIGH 이슈만 보고 (80% 이상 확신)
- 보안, 버그, 설계, 테스트 누락 점검

## 보안 스캔 (Security)

다음과 같은 요청이 감지되면 보안 스캔 모드로 동작합니다:
- "보안 점검", "취약점 스캔", "시크릿 체크"
- "security scan", "vulnerability check"

### 보안 스캔 모드 동작

- 하드코딩된 시크릿, 위험한 기본값 탐지
- 인증/인가 누락, 입력 검증 점검
