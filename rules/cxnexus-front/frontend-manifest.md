# CX Nexus Frontend Harness Manifest

이 문서는 `cxnexus-front`의 프론트엔드 하니스 문서의 진입점입니다. 각 파일은 하나의 역할에 집중하며, 중복 내용은 여기에 남기지 않고 세부 문서로 분리합니다.

## 구조
- `prettier-setup.md` — 코드 포맷팅 및 Prettier 설정
- `frontend-guidelines.md` — 프론트엔드 엔지니어링 원칙과 구현 가이드
- `rtk-query.md` — RTK Query 기반 상태 관리 및 서버 상태 설계
- `design-system.md` — 디자인 시스템, UI 토큰, 컴포넌트 사용 규칙
- `error-handling.md` — 에러 코드 표준, 공통 처리, 로그 및 디버깅

## 사용 방법
1. 먼저 이 매니페스트에서 필요한 영역을 확인합니다.
2. 각 세부 문서에서 해당 주제의 구체적인 규칙과 사례를 참고합니다.
3. 새로운 프론트엔드 가이드가 필요할 경우, 새 파일을 추가하고 이 매니페스트에 링크를 추가합니다.

## 문서 역할의 분리 원칙
- 하나의 문서는 하나의 역할에 집중합니다.
- 공통 개요는 이 매니페스트에만 둡니다.
- 상세 규칙과 예시는 각 주제별 문서에 둡니다.

## 참고
- 이 디렉터리의 문서들은 `frontend` 하니스 실행과 유지보수를 쉽게 하기 위한 목적으로 작성되었습니다.
- 각 문서의 제목과 파일명은 현재 `cxnexus-front`에서 자연스럽게 사용되는 주제 중심 이름으로 정리되어 있습니다.
